# 功能设计方案

## 1. 背景与目标
- **项目背景**：电动示警桩用于园区/道路临时警示与巡检点位移动，当前存在人工搬运效率低、重复作业强、点位调整慢的问题。
- **设计目标**：实现示警桩可控移动、稳定驻停、远程下发与状态回传，并为后续自动驾驶（定位、感知、路径跟踪）预留接口。
- **成功标准**：
  - 样机在平整路面可稳定移动，最大速度约 `1.0 m/s`。
  - 双轮编码器闭环下，直线偏航可控，能完成常规巡逻/挪位任务。
  - 每日两次充电策略下，单段 `5~6h` 任务不断电。
  - 急停、失联保护、过流保护等安全机制可触发并可靠生效。

## 2. 本次设计的核心功能（重点）
### 2.1 功能总览（一期 + 智能增强）
1. 移动控制功能：前进、后退、原地转向、定点停车。
2. 双轮同步功能：基于编码器做左右轮速度闭环，提升直线稳定性。
3. 安全保护功能：急停、失联停车、欠压限速、过流保护。
4. 状态监控功能：实时回传速度、电量、电流、在线状态、故障码。
5. 供电与续航功能：`12V 36Ah` 电池供电，支持每日两次充电节奏。
6. ROS 视觉功能：基于 ROS 进行摄像头感知与视觉控制，支撑巡检与障碍预警。
7. 语音控制功能：支持语音唤醒或语音指令控制小车基本动作。
8. 扬声器报警功能：异常状态语音播报、现场告警与远程喊话。
9. AI 本地推理功能：在 ROS 上位机部署视觉模型进行识别与决策辅助。
10. ROS+单片机协同控制：ROS 负责视觉与决策，STM32 负责底层实时闭环控制。
11. GPS 定位功能：上报车辆经纬度，支持远程查看位置与轨迹。
12. 远程控制功能：通过 4G/WiFi 实现远程操控、状态回传与基础运维。

## 3. 关键功能设计说明
### 3.1 运动控制功能（F01~F03）
- 采用双轮差速结构，控制核心为 `STM32F407VET6`，驱动核心为 `AT8236`。
- 通过编码器采样实现左右轮闭环控制，保障直线行驶稳定性与可重复性。
- 面向现场使用，优先保证低速可控与停车可靠，不追求高速极限。

### 3.2 安全保护功能（F04~F07）
- 建立“急停最高优先级”策略，任何控制命令都不能覆盖急停动作。
- 通信链路失联时采用超时停车机制，防止设备继续无控运行。
- 对电池欠压与电机过流设阈值保护，避免电池过放和驱动器损坏。

### 3.3 状态与运维功能（F08~F10）
- 将速度、电量、故障码作为最小可运维数据集，支持快速定位问题。
- 保证“命令 -> 执行 -> 回传”闭环，便于远程判断设备是否按指令动作。
- 结合 `36Ah` 电池与每日两次充电策略，保证单段任务稳定运行。

### 3.4 扩展预留功能（F11~F12）
- 结构上预留上位计算与传感模块空间，避免后续升级重做底盘。
- 接口上预留电源与通信能力，支持自动驾驶、远程图传与语音交互扩展。
- 一期先把“可动、可控、可保底安全”做扎实，二期再增强“智能化”。

### 3.5 ROS 视觉与 AI 功能（F13、F16）
- 摄像头接入 ROS 上位机后，先实现基础视觉能力：障碍检测、目标区域识别、场景状态判断。
- AI 模型优先选轻量版本（如小型目标检测/分类模型），保证上位机可实时运行。
- 视觉结果不直接越过安全层控制电机，最终运动执行仍由 STM32 安全闭环接管。

### 3.6 语音控制与扬声器报警（F14、F15）
- 语音输入经麦克风采集后解析为标准控制指令，再下发至运动控制模块。
- 设置语音白名单指令（如“前进”“停止”“左转”“右转”），降低误识别风险。
- 扬声器用于低电量、故障、失联等主动报警，也支持远程人工喊话。

### 3.7 ROS、单片机、GPS、远程控制（F17~F19）
- 采用 ROS+STM32 协同架构：ROS 负责视觉/定位/策略，STM32 负责运动控制与安全闭环。
- GPS 模块周期上报经纬度，支持设备定位、轨迹回放和远程调度辅助。
- 远程控制采用 `WiFi + 4G` 双链路思路，本地调试走 WiFi，跨区域运维走 4G。
- 远控命令必须通过底层安全策略校验，急停与失联停车逻辑始终生效。

## 4. 一期不包含功能（边界说明）
- 不包含完整自动驾驶算法（SLAM、全局路径规划、动态避障）。
- 不包含 RTK 高精定位全链路与云端调度平台。
- 不包含量产级工业认证与大规模制造工艺优化。

## 5. 功能验收标准（按功能看结果）
- F01/F02：设备可完成前进、后退、转向、停车，速度档位切换有效。
- F03：直线测试下左右轮同步有效，路径偏航在可接受范围内。
- F04/F05：急停与失联保护可稳定触发，触发后设备立即停止。
- F06/F07：低电量/欠压/过流触发后保护逻辑正确，设备与电源无异常损伤。
- F08/F09：状态数据可查询，控制命令有明确执行结果回传。
- F10：单段 `5~6h` 任务可连续运行，满足每日两次充电策略。

## 6. 后续待确认（影响二期功能）
- ROS 上位机平台选型（`Jetson Orin Nano` / x86 工控机）与存储方案最终定版。
- 单片机软件框架（裸机/FreeRTOS）与 ROS-STM32 通信协议格式（串口/CAN）最终定版。
- 视觉模型类型（目标检测/语义分割）与实时性指标（FPS 目标值）。
- 语音方案（离线关键词识别或在线语音服务）与抗噪策略。
- GNSS 方案（普通 GPS 或 RTK）与 4G 运营商链路质量。
- 远程控制平台形态（Web 控制台/移动端）及权限分级策略。

## 7. 分阶段实现建议（便于落地）
### 阶段 A：先跑通（1~2 周）
- 打通 `F13/F15/F19`：视频采集、基础报警播报、远程看状态与手动控制。

### 阶段 B：增强控制（2~4 周）
- 打通 `F14/F17/F18`：语音控制、单片机任务调度通信、GPS 实时定位上报。

### 阶段 C：AI 尝试（4~6 周）
- 打通 `F16`：在 ROS 上位机部署轻量 AI 模型，形成“视觉识别 -> 告警/辅助控制建议”闭环。

### 8.6 软件栈与版本建议（配套）
- ROS 侧：`Ubuntu 22.04 + ROS 2 Humble`
